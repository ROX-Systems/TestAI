# Security Checklist — `core/ssh-core`

## Обязательные проверки (MUST)

1. **Секреты в логах/трассировках** (SEC-006, T-003)
   - [ ] Автоматические тесты на отсутствие секретов в логах
   - [ ] Проверка redaction в коде (пароли, ключи)

2. **State machine корректность** (SEC-007)
   - [ ] Запрет stdin/resize до состояния READY
   - [ ] Тесты на недопустимые переходы состояний

3. **Host key verification** (SEC-005)
   - [ ] Строгое соблюдение политик (strict/accept-new/ask)
   - [ ] Тесты на блокировку при изменённом ключе

4. **Безопасное хранение ключей**
   - [ ] Использование `secrecy` для временного хранения
   - [ ] Zeroize памяти после использования

5. **Обработка ошибок**
   - [ ] Все ошибки соответствуют SRS §12
   - [ ] Нет паник в публичном API

## Рекомендуемые проверки (SHOULD)

- Fuzz-тестирование парсинга протокола
- Проверка уязвимостей в зависимостях (cargo audit)
- Интеграционные тесты с разными SSH-серверами
